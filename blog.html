What is document object model(DOM)?

Introduction Document Object Model (DOM) For Web Documents It describes the structure of a document (whether HTML or XML) as an object tree. All content within the document becomes a node in this tree and scripts (usually JavaScript) can then alter its appearance, or manipulate it as necessary.

What is event delegation in the context of the DOM,and why is it useful?

Event delegation is a convention of handling events in DOM where you bind an event to the parent element and all its children inherit that particular listener. The parent listener listens to events triggered by its child elements. Performance Gain: Always attaching listeners on each child can be bad for the application as it increases memory usage so one listener per type to a parent is better. Dynamic Elements: If new child elements are added dynamically (e.g. via AJAX or user interactions), they automatically inherit the parent event listenser and so you don't need to add listeners for them again Cleaner and More Readable Code: Your code may look cleaner, as you have less event listeners to care for. Consistency: Because you are addressing events at the single place,so it make sure that same logic will be applied to all relevant child elements.

CSS classes: The classList property contains the on (to add) and togg le (to toggle whether a CSS is added or not) methods for working with CSS classes in an element. add: Adds specified class to the element. remove: It removes a specific class from the element. toggle: Adds or removes a class based on its presence. cssText Property: Using this property, you can combine several CSS styles as a single string and assign them to one or more stylesheet.
